Pick an Approach

Data structure + rendering strategy for O(1) character lookups

A

Hybrid Bitmask + Run-Length Spans (Recommended)

Data: Uint16Array with 16-bit flags per character — single bitwise AND for O(1) lookup. Sparse annotation index for syllable boundaries & stress spans.

Render: Group consecutive same-class characters into run-length spans (~1.5x current DOM count). CSS class toggling via data-attribute — no React re-render on toggle.

Memory: ~248 KB for 5000 words. Zero-copy ArrayBuffer transfer from worker.

Pros

  • True O(1): single bitwise AND per char
  • Zero-copy worker transfer
  • CSS toggle = no React re-render
  • Follows existing SyntaxSets pattern
  • Incremental: phonemic first, rhythmic later

Cons

  • Bitmask harder to debug than objects
  • Run-length encoding adds code complexity
  • Digraph detection (sh, th, ng) needs care
B

Word Map + Character Offset Table

Data: Per-word phoneme entries with character classification arrays. Uint32Array index maps char positions to word entries.

Render: Same run-length spans. Word-level cache reuses phoneme lookups across edits (Zipf's law: ~3000 unique words cover 95% of tokens).

Memory: ~820 KB for 5000 words. Structured clone transfer (no zero-copy).

Pros

  • Rich queryable structure
  • Excellent word-level caching
  • Easy to debug (readable objects)
  • Mirrors existing SongAnalysis pattern

Cons

  • 13x more memory than bitmask
  • No zero-copy transfer
  • More GC pressure from object allocations
C

Canvas Underline Overlay

Data: Same bitmask array. Render: Zero additional DOM nodes — paint colored underlines on a Canvas behind the text. GPU-accelerated.

Memory: Minimal (~60 KB). But requires character position measurement on every content/font/resize change.

Pros

  • Zero DOM nodes added
  • Could coexist with syntax mode
  • GPU-accelerated painting

Cons

  • Character position measurement is fragile
  • Breaks with ligatures, CJK, emoji
  • No hover interactions per-character
  • Fundamentally different rendering paradigm
